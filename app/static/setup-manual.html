トークン期限、長期トークン更新条件、Threads App secret の場所、HTTPS Redirect URL の必要性は Meta 公式ドキュメントと Meta 公式サンプルを基準にしています。

前提

最初に理解しておく3つ

1

このアプリは「ローカル運用画面」です

普段の操作はローカルでできますが、Threads OAuth の Redirect URI と Webhook は HTTPS が必要です。

2

OAuth は HTTPS の URL が必要です

Meta 公式サンプルでも `localhost` を使わず、ローカル用ドメインを用意して HTTPS 化しています。`http://127.0.0.1` のままでは OAuth は通りません。

3

このアプリは `/callback` で自動受け取りします

認証後に Redirect URL へ戻ると、このアプリが `code` を自動で取り込みます。`認証コードを交換` は、うまく戻らなかったときの予備導線です。

4

GAS を使う場合は「公開窓口」だけ GAS に切り分けます

普段の画面操作はローカルアプリのままです。Meta からの callback だけ GAS Webアプリで受け、ローカルの Threads運用センター があとから `GAS同期` で取り込みます。

Phase 1

Meta 側で先にやること

1

Meta for Developers にログインする

ブラウザで Meta for Developers を開き、Threads API 用のアプリ設定画面を開きます。Meta 側の画面表記は更新で少し変わることがありますが、`Threads API` に関する設定ページを開くのが目的です。

2

Threads App ID と Threads App secret を確認する

Meta 公式ドキュメントでは、長期トークン交換時に `Threads app secret` を使うと明記されています。通常のアプリID/Secretと混同せず、Threads 用の値を控えてください。

3

Meta のカスタマイズ画面で 3 本の URL を登録する

完成版運用では `Redirect Callback URL`、`Deauthorize Callback URL`、`Data Deletion Request URL` を設定します。ローカル直結なら `https://your-domain.example/callback` 形式、GASブリッジなら `https://script.google.com/macros/s/xxxxx/exec?action=oauth_callback`、`...?action=meta_deauthorize`、`...?action=meta_data_deletion` 形式で入れます。`localhost` と `127.0.0.1` は使いません。

4

最初に使う権限を決める

このアプリでは、まず `threads_basic`、`threads_content_publish`、`threads_manage_insights`、`threads_manage_replies`、`threads_read_replies` を入れる前提です。長期トークン更新には `threads_basic` が重要です。

5

Webhook を使うなら公開 HTTPS URL を用意する

自動返信で Webhook を使う場合は、Meta から到達できる公開 HTTPS URL が必要です。あとで `WebhookベースURL`、`Verify Token`、`Webhook Secret` に入れます。

Phase 3

このアプリで実際に押す順番

1

`設定` を開いて保存する

左メニューの `設定` を押します。最初は初期値のままでも大丈夫なので、`設定を保存` を押して保存してください。

2

`CLIを検出` を押す

Claude Code または Codex のうち使いたいものが `利用可` か確認します。`未検出` のときは、先にそのCLIをPC側でインストール・ログインしてからやり直します。テーブルの「動作確認」ボタンで PONG 応答を確認できれば本番運用可能です。

2.5

GAS を使うなら `設定` に GAS 情報を入れる

`GAS WebアプリURL`、`GAS共有キー`、`GASブリッジを有効にする` を設定してから、もう一度 `設定を保存` を押します。Meta に入れる URL は GAS ブリッジ側で生成された exec URL（`https://script.google.com/macros/s/xxxxx/exec` 形式）に揃えます。

3

`アカウント` を開いて基本情報を入れる

`表示名`、`ユーザー名`、`発信ジャンル`、`運用モード` を入れてから、一度 `アカウントを保存` を押します。

4

同じ画面で Threads 連携情報を入れる

`Threads App ID`、`Threads App Secret`、`Redirect URI`、`Scopes` を入れます。Webhookを使う予定なら `Verify Token` と `Webhook Secret` も入れます。`Deauthorize` と `Data Deletion` の 2 本は、アプリ保存後に `連携URLガイド` に表示された URL を Meta 側に登録します。入力したら再度 `アカウントを保存` を押します。

5

`OAuth URLを開く` を押す

ブラウザで Meta の認証画面が開きます。許可まで進めてください。

6

完了画面が開くか確認する

認証後に `/callback` の完了画面が開けば、自動で短期トークン取得、長期トークン交換、プロフィール取得まで進みます。

7

うまく戻らないときだけ `認証コードを交換` を使う

ブラウザの URL に `?code=...` が残っている場合だけ、その `code` をコピーして `認証コードを交換` に貼り付けます。

8

`メトリクス取得` を押して接続確認する

エラーが出ず、アカウント一覧の認証状態が `正常` に近い表示になれば初期接続は成功です。

このアプリ固有

初心者向けの現実的な運用方法

今は「公開HTTPS URLを1つ用意する」方法が一番分かりやすいです

このアプリは普段はローカルで動きますが、OAuth、Webhook、Deauthorize、Data Deletion は HTTPS が必要です。初心者なら、まずは HTTPS の公開URLを1つ用意し、その URL を `Redirect URI` と `WebhookベースURL` に使う構成が分かりやすいです。

値 例 どこに入れるか
公開HTTPS URL `https://abc123.example.com` `設定 > WebhookベースURL`
Redirect URI `https://abc123.example.com/callback` `アカウント > Redirect URI`
Meta 側の Redirect Callback URL `https://abc123.example.com/callback` Meta App Dashboard
Meta 側の Deauthorize Callback URL `https://abc123.example.com/api/meta/deauthorize` Meta App Dashboard
Meta 側の Data Deletion Request URL `https://abc123.example.com/api/meta/data-deletion` Meta App Dashboard
